# 완료매물

## 페이지 목적

- 관리 매물중 거래가 완료된 매물만 모아서 조회하는 페이지
- 완료된 계약의 이력, 고객 정보, 정산 내역 등을 한눈에 확인하는 것이 목적
- 완료된 매물을 기준으로 통계, 리포트, 고객 관리 작업을 할 수 있어야 함

## 필요한 이유

- 현재 관리매물 화면은 진행중인 매물이 중심이라 완료된 매물 관리가 어려움
- 완료된 매물만 별도로 모아서 보면 업무 효율이 높아짐
- 완료된 계약의 고객 정보를 기반으로 후속 관리나 재계약 기회를 찾을 수 있음
- 완료된 매물의 통계 데이터를 활용해 업무 성과를 분석할 수 있음

## 1차 구현 범위

- 별도 완료매물 테이블 없이 `ManagedProperty` 기반으로 완료된 매물만 조회
- 완료 상태는 `status` 필드를 기준으로 판단
- 완료된 계약의 기본 정보와 고객 정보를 표시
- 완료된 매물에 대한 통계 요약 제공
- 향후 필요 시 별도 완료매물 테이블로 분리 가능
- 목록을 매매와 임대로 분리

## 화면 구성


### 1. 완료매물 그리드 영역

- 완료된 매물 목록 표시
- 매물별로 거래 정보와 고객 정보 표시
- 기본은 모든 완료 매물 표시
- 필터 적용 시 조건에 맞는 매물만 표시

## 2. 그리드 구조

- 매물명
- 건물명
- 책임자
- 책임자 전화번호
- 주소
- 거래유형
- 거래 대상자 명
- 거래 대상자 전화번호
- 거래 금액
- 거래일
- 계약 기간
- 만기일
- 상태
- 메모
- 빠른 액션
  - 상세 보기
  - 고객 정보 보기
  - 통계 보기



### 완료 상태 판단

- `status` 필드가 `완료` 또는 `종료`인 경우 완료 매물로 취급
- `contractEndDate`가 현재 날짜보다 과거인 경우 완료 매물로 취급
- 필요 시 여러 완료 상태를 포함하도록 확장 가능

### 거래 금액 표시

- 매매: 거래 금액 표시
- 전세: 전세 보증금 표시
- 월세: 보증금 + 월세 표시
- 단위는 `만원`으로 통일
- 통화 기호 `₩` 표시

### 거래 기간 계산

- `contractEndDate` - `contractDate`로 계산
- 일 단위로 표시 (예: 730일)
- 필요 시 월 단위로 변환해서 표시

### 고객 정보 표시

- `tenantName`, `tenantPhone` 사용
- 매매의 경우 `managerName`, `managerPhone`도 표시 가능
- 고객 정보가 없으면 `정보 없음` 표시

## 완료매물 상세 보기

### 현재 스키마 기준

- 현재 `ManagedProperty`에는 아래 필드가 이미 존재함
  - `articleName`
  - `buildingName`
  - `address`
  - `propertyType`
  - `contractType`
  - `contractDate`
  - `contractEndDate`
  - `tenantName`
  - `tenantPhone`
  - `managerName`
  - `managerPhone`
  - `notes`
  - `status`

### 현재 구조의 장점

- 별도 완료매물 테이블 없이 기존 데이터로 구현 가능
- `status` 필드를 활용해 완료 상태 쉽게 판단
- 계약 관련 정보가 이미 잘 정리되어 있음

### 권장 구현 순서

### 1단계

- 완료된 매물 목록 화면 구현

### 2단계

- 완료매물 상세 보기 추가
- 메모 표시
