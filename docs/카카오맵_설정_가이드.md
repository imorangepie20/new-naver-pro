# 카카오맵 설정 가이드

## 1) 필수 환경 변수

`.env.development`에 아래 값을 설정합니다.

```env
VITE_KAKAO_MAP_APP_KEY=카카오_JavaScript_키
KAKAO_REST_API_KEY=카카오_REST_API_키
```

- `VITE_KAKAO_MAP_APP_KEY`: 프론트 지도 SDK 로딩용
- `KAKAO_REST_API_KEY`: 서버 주소->좌표 변환(지오코딩)용

## 2) 카카오 개발자 콘솔 설정

- 앱 선택
- `플랫폼 > Web`에 도메인 등록
- 예시:
  - `http://localhost:5173`
  - `http://imapplepie20.tplinkdns.com:5173`

주의:
- 프로토콜(`http/https`)과 포트(`:5173`)까지 정확히 일치해야 합니다.
- 등록 후 반영까지 수 분이 걸릴 수 있습니다.

## 3) 현재 프로젝트의 좌표 변환 방식

- 지도 표시: 브라우저에서 카카오 SDK 직접 로드
- 사무실 주소 좌표 계산: 서버에서 카카오 Local REST API 호출

## 4) 자주 발생하는 오류

### `domain mismatched! caller=...`

원인:
- 카카오 콘솔 Web 도메인 미등록 또는 오탈자

해결:
- 오류 메시지의 `caller` 값을 그대로 콘솔 Web 도메인에 추가

### `address.json 401 (Unauthorized)`

원인:
- 서버 `KAKAO_REST_API_KEY` 누락/오입력
- REST API 키가 다른 카카오 앱의 키

해결:
- 카카오 콘솔에서 REST API 키 재확인 후 `.env.development`에 `KAKAO_REST_API_KEY` 설정
- 서버 재시작

### `Kakao Maps constructors are unavailable after SDK load`

원인:
- SDK 로드 실패 또는 도메인 검증 실패
- 확장 프로그램/보안 정책으로 스크립트 차단

해결:
- 브라우저 Network 탭에서 `t1.daumcdn.net/.../kakao.js` 상태 확인
- 광고 차단/보안 확장 비활성화 후 재시도
- 강력 새로고침 후 재확인

## 5) 재시작 체크리스트

설정 변경 후 아래를 수행합니다.

1. 백엔드 재시작
2. 프론트엔드 재시작
3. 브라우저 강력 새로고침
