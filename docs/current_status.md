## 1. 프로젝트 개요

이 프로젝트는 부동산 실무용 웹 관리 도구를 중심으로 확장 중인 통합 관리자 시스템이다. 현재 실사용 핵심은 아래 4개 영역이다.

- 부동산 매물 검색/등록/관심/관리
- 주소 기반 시세 및 REB 통계 분석
- 일정 관리 캘린더
- 실무형 대시보드

현재는 웹앱 중심으로 계속 고도화 중이며, 데스크탑 앱은 아직 시작하지 않았다.  
데스크탑 앱은 웹 구조와 데이터 구조를 먼저 정리한 뒤 별도 프로젝트로 시작하는 방향으로 판단한 상태다.

## 3. 현재 구조 판단

### 현재 상태
- 웹 중심 구조는 계속 유지
- 서버는 `src/server/index.ts` 단일 진입점 기반
- 프론트는 `src/App.tsx` 라우트 기준으로 페이지 구성
- 사용자 인증은 JWT + `authFetch` 기반
- 정적 빌드 결과물은 `dist/`

### 현재 판단된 구조 방향
- 앞으로는 `중앙DB + 로컬DB` 이원 구조가 더 적합하다는 결론
- 현재 프로젝트에서 Prisma는 절차 비용이 커서 손해라는 판단
- 우선순위는 다음 순서로 정리됨
  1. 웹 프로젝트를 중앙DB/로컬DB 구조에 맞게 재정리
  2. Prisma 제거 또는 최소화
  3. 웹 최적화
  4. 그 후 데스크탑 앱 프로젝트 시작

### 아직 시작하지 않은 항목
- 중앙DB/로컬DB 이원화 구현
- Prisma 제거 작업
- 서버 모니터링 데몬
- 데스크탑 앱 프로젝트 시작

- 서버 모니터링/프로세스 관리가 아직 정식 아님

### 기능 리스크
- 카카오 알림톡 실발송은 아직 최종 연결 전
- 카카오 API rate limit 발생 가능
- 대시보드는 계속 다듬을 여지가 있음

### 빌드 상태
- `npm run build`는 통과
- 기존 경고는 유지
  - `onnxruntime-web`의 `eval` 경고
  - 번들 크기 경고

## 12. 현재 우선순위 판단

현재 가장 현실적인 우선순위는 아래 순서다.

5. 서버 모니터링 도입
6. 그 다음 데스크탑 앱 프로젝트 시작
